    ü§î ¬øQu√© es React Router y por qu√© lo necesitamos?

    El problema

    React es una SPA (Single Page Application) - toda la aplicaci√≥n vive en una sola p√°gina HTML. Pero los usuarios esperan: - URLs diferentes para diferentes p√°ginas - Bot√≥n atr√°s/adelante funcional - Poder compartir enlaces - Marcar p√°ginas como favoritos

    La soluci√≥n: React Router

    React Router permite crear la ilusi√≥n de m√∫ltiples p√°ginas en tu SPA:

    Sin React Router:               Con React Router:
misitioweb.com                  misitioweb.com/
(Todo en una p√°gina)            misitioweb.com/productos
                                misitioweb.com/productos/123
                                misitioweb.com/contacto

    Conceptos clave

    // Navegaci√≥n tradicional (recarga toda la p√°gina)

import { useParams } from 'react-router-dom'

function ProductoDetalle() {

    // Cargar producto usando el ID

      <h1>{producto.nombre}</h1>
      <p>ID del producto: {id}</p>
    </div>
  )

    Breadcrumbs din√°micos

    import { useLocation, Link } from 'react-router-dom'

function Breadcrumbs() {
  const location = useLocation()

  const pathnames = location.pathname.split('/').filter(x => x)

  return (
    <nav aria-label="breadcrumb">
      <ol className="breadcrumb">
        <li>
          <Link to="/">Inicio</Link>
        </li>

        {pathnames.map((name, index) => {
          const routeTo = `/${pathnames.slice(0, index + 1).join('/')}`
          const isLast = index === pathnames.length - 1

          return (
            <li key={name}>
              {isLast ? (
                <span>{name}</span>
              ) : (
                <Link to={routeTo}>{name}</Link>
              )}
            </li>
          )
        })}
      </ol>
    </nav>
  )
}
